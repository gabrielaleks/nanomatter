# Brainstorming nanomatter

I already have Home Assistant running on my homelab, managing some matter devices on my local network. Now I’m building my own matter controller on the same homeserver, using matter.js on my raspberry pi. Since my controller and Home Assistant are on the same network, both of them can reach the devices directly over ip.

In matter, a device can be controlled by multiple controllers at the same time, thanks to multi-admin support. This means I can add my own controller without interfering with Home Assistant. The easiest way for me to do this is to first commission a new device (like a lamp) using Home Assistant. Home Assistant becomes the first controller and fabric administrator of the device. Once the device is commissioned with Home Assistant, it trusts that fabric and is ready to accept other controllers.

To add my own controller, I can use a multi-admin commissioning code generated by Home Assistant. This is different from the original setup code that comes with the device. Using that shared code, my matter.js controller can perform its own commissioning, join the device’s fabric as a secondary admin and start interacting with it directly. After this, I can read its state, subscribe to updates and issue commands independently of Home Assistant. Home Assistant doesn’t act as a bridge here - both controllers talk directly to the device.

I could also choose to commission the device directly with my own controller, without going through Home Assistant. That would be harder though. Many consumer devices initially rely on Bluetooth Low Energy (BLE) for commissioning, or they require Thread connectivity through a border router. Since my Raspberry Pi controller may not have BLE access or Thread support, it can’t always commission devices directly over these initial channels. Also, the tools and SDK support for first-time commissioning on a backend controller are less mature than what Home Assistant provides, which handles everything automatically.

Direct commissioning with my controller isn’t impossible, though. I could implement it later by giving my controller full commissioner capabilities: it would need to handle device discovery, open a commissioning window, establish a secure PASE session, exchange certificates and provision network credentials over Wi-Fi or Thread. That would allow my controller to become the first fabric admin for a device, completely bypassing Home Assistant.

For now, the practical approach for me is to commission the device first with Home Assistant, then add my controller using the multi-admin process. This way I get full access to the device without dealing with BLE or Thread commissioning complications. Later, I could extend my controller to handle first-time commissioning on its own if I want to.
